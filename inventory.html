Inventory Database Structure

The Inventory module in Magento 2 manages stock levels, availability, and multi-source inventory. Magento 2.3 introduced a completely new inventory management system called MSI (Multi-Source Inventory), which allows products to be stocked in multiple physical locations.

Legacy vs. MSI

Magento 2 has two inventory systems:

  • Legacy Inventory - Uses cataloginventory_* tables (Magento 2.0-2.2)
  • MSI (Multi-Source Inventory) - Uses inventory_* tables (Magento 2.3+)

Both systems can coexist in Magento 2.3+, with MSI being the primary system.

Legacy Inventory Tables

cataloginventory_stock

Defines inventory stocks. In the legacy system, there's typically only one stock (ID 1).

Column Type Description
stock_id smallint Primary key
website_id smallint Website ID
stock_name varchar Stock name

cataloginventory_stock_item

Stores inventory data for each product. This is the main inventory table in the legacy system.

Column Type Description
item_id int Primary key
product_id int Foreign key to catalog_product_entity.entity_id
stock_id smallint Foreign key to cataloginventory_stock.stock_id
qty decimal Quantity in stock
is_in_stock smallint Whether the product is in stock (1=yes, 0=no)
manage_stock smallint Whether to track inventory for this product
use_config_manage_stock smallint Whether to use system config for manage_stock
min_qty decimal Minimum quantity allowed in cart
max_sale_qty decimal Maximum quantity allowed in cart
backorders smallint Backorders status (0=no, 1=yes, 2=notify)
low_stock_date timestamp When the product reached low stock
notify_stock_qty decimal Quantity threshold for low stock notification

cataloginventory_stock_status

Indexed stock status for faster catalog browsing. This is a denormalized table for performance.

Column Type Description
product_id int Foreign key to catalog_product_entity.entity_id
website_id smallint Website ID
stock_id smallint Foreign key to cataloginventory_stock.stock_id
qty decimal Quantity in stock
stock_status smallint Stock status (1=in stock, 0=out of stock)

Multi-Source Inventory (MSI) Tables

inventory_source

Defines inventory sources (physical locations where products are stored).

Column Type Description
source_code varchar Primary key (unique source code)
name varchar Source name
enabled smallint Whether the source is enabled
description text Source description
latitude decimal Geographical latitude
longitude decimal Geographical longitude
country_id varchar Country code
region_id int Region/state ID
region varchar Region/state name
city varchar City
street varchar Street address
postcode varchar Postal code
contact_name varchar Contact person name
email varchar Contact email
phone varchar Contact phone

inventory_source_item

Stores inventory data for each product at each source.

Column Type Description
source_item_id int Primary key
source_code varchar Foreign key to inventory_source.source_code
sku varchar Product SKU
quantity decimal Quantity in stock at this source
status smallint Stock status at this source (1=in stock, 0=out of stock)

inventory_stock

Defines inventory stocks. In MSI, a stock is a collection of sources that are assigned to sales channels.

Column Type Description
stock_id int Primary key
name varchar Stock name

inventory_source_stock_link

Maps sources to stocks (many-to-many relationship).

Column Type Description
link_id int Primary key
stock_id int Foreign key to inventory_stock.stock_id
source_code varchar Foreign key to inventory_source.source_code
priority smallint Source priority within the stock (for shipping)

inventory_stock_sales_channel

Maps stocks to sales channels (websites).

Column Type Description
type varchar Sales channel type (e.g., 'website')
code varchar Sales channel code (e.g., website code)
stock_id int Foreign key to inventory_stock.stock_id

inventory_reservation

Tracks temporary inventory reservations (e.g., items in shopping carts).

Column Type Description
reservation_id int Primary key
stock_id int Foreign key to inventory_stock.stock_id
sku varchar Product SKU
quantity decimal Reserved quantity (negative value)
metadata varchar JSON metadata about the reservation
created_at timestamp When the reservation was created

Inventory Index Tables

inventory_stock_item

Stores inventory configuration for each product.

Column Type Description
sku varchar Product SKU
stock_id int Stock ID
quantity decimal Total quantity across all sources
is_salable smallint Whether the product is salable

Data Analysis Tips

When analyzing inventory data:

  • For Magento 2.3+, use the MSI tables for the most accurate inventory data
  • To get the total quantity of a product across all sources: 
    SELECT sku, SUM(quantity) as total_qty
FROM inventory_source_item
WHERE sku = '[product_sku]'
GROUP BY sku
  • To get inventory by source: 
    SELECT s.name as source_name, si.sku, si.quantity
FROM inventory_source_item si
JOIN inventory_source s ON si.source_code = s.source_code
WHERE si.sku = '[product_sku]'
  • For legacy inventory data: 
    SELECT product_id, qty, is_in_stock
FROM cataloginventory_stock_item
WHERE product_id = [product_id]
