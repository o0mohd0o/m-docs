Customization Techniques

Plugins (Interceptors)

Plugin Types

TypeMethod PrefixWhen ExecutedCan Modify Result
BeforebeforeBefore the original methodNo (can modify arguments)
AroundaroundWraps the original methodYes

Before Plugin Example

<?php
namespace Vendor\Module\Plugin;

class ProductPlugin
{
    public function beforeSetName(
        \Magento\Catalog\Model\Product $subject,
        $name
    ) {
        // Modify the name argument before it's set
        $name = strtoupper($name);
        return [$name];
    }
}

After Plugin Example

<?php
namespace Vendor\Module\Plugin;

class ProductPlugin
{
    public function afterGetName(
        \Magento\Catalog\Model\Product $subject,
        $result
    ) {
        // Modify the return value
        return $result . ' - Modified';
    }
}

Around Plugin Example

<?php
namespace Vendor\Module\Plugin;

class ProductPlugin
{
    public function aroundGetName(
        \Magento\Catalog\Model\Product $subject,
        \Closure $proceed
    ) {
        // Code before original method
        $result = $proceed(); // Call original method
        // Code after original method
        return 'Prefix ' . $result . ' Suffix';
    }
}

use Magento\Framework\Event\Observer;

Dispatching Custom Events

<?php
// In your class constructor
public function __construct(
    \Magento\Framework\Event\ManagerInterface $eventManager
) {
    $this->eventManager = $eventManager;
}

// Dispatch event
$this->eventManager->dispatch('custom_event_name', [
    'product' => $product,
    'custom_data' => $data

Retrieving Config Values

<?php
use Magento\Framework\App\Config\ScopeConfigInterface;
use Magento\Store\Model\ScopeInterface;

public function __construct(
    ScopeConfigInterface $scopeConfig
) {
    $this->scopeConfig = $scopeConfig;
}

public function getConfigValue()
{
    return $this->scopeConfig->getValue(
        'vendor_module_settings/general/enabled',
        ScopeInterface::SCOPE_STORE
    );
}

CLI Commands

Command Class

<?php
namespace Vendor\Module\Console\Command;

use Symfony\Component\Console\Command\Command;
use Symfony\Component\Console\Input\InputInterface;
use Symfony\Component\Console\Output\OutputInterface;
use Symfony\Component\Console\Input\InputArgument;

class CustomCommand extends Command
{
    const NAME_ARGUMENT = 'name';

    protected function configure()
    {
        $this->setName('vendor:custom:command')
             ->setDescription('Custom command description')
             ->addArgument(
                 self::NAME_ARGUMENT,
                 InputArgument::REQUIRED,
                 'Name argument'
             );
        
        parent::configure();
    }

    protected function execute(InputInterface $input, OutputInterface $output)
    {
        $name = $input->getArgument(self::NAME_ARGUMENT);
        $output->writeln("Hello, $name!");
        return 0;
    }
}

  • Know when to use plugins vs observers vs preferences
  • Understand plugin execution order and priorities
  • Remember that around plugins can prevent method execution
  • Observers can't modify return values, only side effects
  • Preferences should be last resort - they can cause conflicts
