CMS & Content Database Structure

The CMS (Content Management System) module in Magento 2 manages static content such as pages, blocks, and widgets. This module allows store administrators to create and manage content without requiring technical knowledge.

Key Relationships

Note: This is a simplified diagram showing the main relationships.

CMS Page Tables

cms_page

Stores CMS pages such as the homepage, about us, terms and conditions, etc.

Column Type Description
page_id smallint Primary key
title varchar Page title
page_layout varchar Page layout (e.g., 1column, 2columns-left)
meta_keywords text Meta keywords for SEO
meta_description text Meta description for SEO
identifier varchar URL key (e.g., 'about-us')
content_heading varchar Content heading
content mediumtext Page content (HTML)
creation_time timestamp When the page was created
update_time timestamp When the page was last updated
is_active smallint Whether the page is active
sort_order smallint Sort order
layout_update_xml text Custom layout update XML
custom_theme varchar Custom theme
custom_root_template varchar Custom root template

cms_page_store

Maps CMS pages to store views (many-to-many relationship).

Column Type Description
page_id smallint Foreign key to cms_page.page_id
store_id smallint Store view ID (0 for all stores)

CMS Block Tables

cms_block

Stores CMS blocks, which are reusable content fragments that can be placed in various locations.

Column Type Description
block_id smallint Primary key
title varchar Block title
identifier varchar Block identifier (used in layout XML)
content mediumtext Block content (HTML)
creation_time timestamp When the block was created
update_time timestamp When the block was last updated
is_active smallint Whether the block is active

cms_block_store

Maps CMS blocks to store views (many-to-many relationship).

Column Type Description
block_id smallint Foreign key to cms_block.block_id
store_id smallint Store view ID (0 for all stores)

Widget Tables

widget_instance

Stores widget instances, which are dynamic content elements that can be placed in various locations.

Column Type Description
instance_id int Primary key
instance_type varchar Widget type (PHP class)
theme_id int Theme ID
title varchar Widget title
store_ids varchar Comma-separated list of store IDs
widget_parameters text Widget parameters (serialized)
sort_order smallint Sort order

widget_instance_page

Defines which pages a widget appears on.

Column Type Description
page_id int Primary key
instance_id int Foreign key to widget_instance.instance_id
page_group varchar Page group (e.g., 'all_pages', 'specific_pages')
layout_handle varchar Layout handle
block_reference varchar Block reference (container)
page_for varchar Page type
entities text Specific entities (e.g., page IDs)
page_template varchar Page template

widget_instance_page_layout

Maps widget instances to page layouts.

Column Type Description
page_id int Foreign key to widget_instance_page.page_id
layout_update_id int Layout update ID

URL Rewrite Tables

url_rewrite

Stores URL rewrites for products, categories, CMS pages, and custom URLs.

Column Type Description
url_rewrite_id int Primary key
entity_type varchar Entity type (product, category, cms-page, custom)
entity_id int Entity ID
request_path varchar Request path (URL key)
target_path varchar Target path (internal path)
redirect_type smallint Redirect type (0=no redirect, 301=permanent, 302=temporary)
store_id smallint Store view ID
description varchar Description
is_autogenerated smallint Whether the rewrite was automatically generated
metadata varchar Additional metadata (JSON)

Design Tables

design_config_grid_flat

Stores design configuration for different store views.

Column Type Description
entity_id int Primary key
store_website_id int Website ID
store_group_id int Store group ID
store_id int Store view ID
theme_theme_id varchar Theme ID

theme

Stores theme information.

Column Type Description
theme_id int Primary key
parent_id int Parent theme ID
theme_path varchar Theme path
theme_title varchar Theme title
preview_image varchar Preview image path
is_featured tinyint Whether the theme is featured
area varchar Area (frontend, adminhtml)
type smallint Theme type
code text Theme code

Data Analysis Tips

When analyzing content data:

  • To get all active CMS pages: 
    SELECT p.page_id, p.title, p.identifier, p.content, ps.store_id
FROM cms_page p
JOIN cms_page_store ps ON p.page_id = ps.page_id
WHERE p.is_active = 1
  • To get all active CMS blocks: 
    SELECT b.block_id, b.title, b.identifier, b.content, bs.store_id
FROM cms_block b
JOIN cms_block_store bs ON b.block_id = bs.block_id
WHERE b.is_active = 1
  • To get URL rewrites for a specific product: 
    SELECT request_path, target_path, redirect_type, store_id
FROM url_rewrite
WHERE entity_type = 'product' AND entity_id = [product_id]
  • To find widgets used on a specific page: 
    SELECT wi.instance_id, wi.instance_type, wi.title, wip.block_reference
FROM widget_instance wi
JOIN widget_instance_page wip ON wi.instance_id = wip.instance_id
WHERE wip.layout_handle = 'cms_page_view' AND wip.entities LIKE '%[page_id]%'
